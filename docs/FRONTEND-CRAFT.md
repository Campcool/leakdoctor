# 前台質感精修指南（Claude → Codex）

> 建立：2026-07-12（Claude）。這份是「**怎麼做到高質感**」的技法手冊，補足 `DESIGN.md`（規範「該長怎樣」）沒寫的執行細節。
> 目標：把首頁與四大服務頁做到 **Apple / 小米 / 三星** 那種科技大牌質感——大膽自信的排版、克制的用色、真實有溫度的影像、精緻的互動——而不是「通用線性圖示 + 藍紫漸層 + 全套陰影 + 零互動」的 AI 樣板感。
> 用法：Codex 逐頁、逐區塊慢慢套用；每套一項就對照 §9 檢查表。**不需要一次到位**。

---

## 1. 先誠實診斷：現在為什麼還有「AI 感」

對照目前 `index.html`，可辨識的 AI 樣板特徵（依 `artifact-design` 技能整理）：

1. **emoji 當服務圖示**（❄️🧺🧽💧📖🔧🛡️）—— 科技大牌 UI 絕不用 emoji。
2. **通用線性 icon**：可被任何網站替換、沒有品牌個性 = 樣板感。這是本次業主指出的重點。
3. **藍紫漸層 Hero**：AI 網站最典型的樣板之一。
4. **每張卡都套陰影 + 大圓角**：資訊靠陰影切割 = 廉價感。DESIGN.md 已規定「優先用邊框，不依賴陰影」。
5. **字太小太密**（桌機 body 15px、標題 1.65rem）：大牌都用**大而自信**的字。
6. **幾乎零互動**：只有卡片 hover 微浮，沒有捲動進場、產品切換、微互動。

> Claude 已做一版首頁原型示範方向（互動服務切換器、捲動進場、深淺色、專屬色）。本檔把「原型用到的技法」與「還可以再上一層的作法」完整寫下來。

## 2. 圖示（Icon）策略——做出「不像 AI」的圖示

「不像 AI」的關鍵不是「換一套線圖」，而是 **光學一致性 + 品牌母題 + 適當的份量感**。

### 2.1 選一套授權圖庫當基底（別自己東拼西湊）

| 圖庫 | 授權 | 風格 | 適合 |
|---|---|---|---|
| **Lucide** | ISC（商用免費） | 乾淨線性、社群活躍 | 一致性最高，當基底最穩 |
| **Phosphor** | MIT | 提供 6 種粗細＋填色/duotone | 想要 duotone 重點色時 |
| **Tabler Icons** | MIT | 線性、4000+ | 補齊冷門圖示 |
| **Iconoir / Heroicons** | MIT | 精緻線性 | 備援 |

> 版權紅線：**不可**直接抓 Apple / 小米 / 三星或任何他站的 icon/圖片來用（著作權）。只用上表這種明確授權的來源，或自己畫。

### 2.2 讓 icon 一致（消除「拼湊感」）

- **統一網格**：全部 24×24 viewBox、`stroke-width` 固定（線性建議 1.6–1.8）、`stroke-linecap/linejoin=round`、圓角一致。
- **統一光學份量**：不同 icon 的「視覺重量」要一致——複雜的圖（洗衣機）不要比簡單的圖（水滴）看起來重太多，必要時微調細節數量。
- **用 `currentColor`**：SVG 內不寫死顏色，靠 CSS `color` 控制，才能一鍵套用專屬色與深淺色。
- **一個檔案管理**：做成 `<symbol>` sprite（一份 `icons.svg`，頁面用 `<svg><use href="/icons.svg#aircon"/></svg>`），避免每頁複製長 path。

### 2.3 從「通用」升級到「品牌化」（這步最關鍵）

- **加入專屬母題**：冷氣＝出風/氣流線；洗衣機＝旋轉水流；居家清潔＝泡泡/光澤；漏水＝水滴+聲納定位圈。讓四個 icon 一看就對應到灰汰郎的服務，而不是隨便的家電圖。
- **duotone / 填色重點**：純線性容易單薄。用「線 + 一塊品牌色填色」（Phosphor duotone 風），或線性 + 專屬色的小圓點/高光，立刻有設計感。
- **兩種尺寸兩種畫法**：小尺寸（16–20px 導覽）用精簡線性；大尺寸（Hero/卡片 40–100px）可用更精緻的填色或雙色版本，不要只是把小圖放大。
- **微動效**：hover 或進場時 icon 動一下（氣流線流動、水滴滴落、泡泡浮起）——用 CSS `@keyframes` 或 SVG SMIL；一律尊重 `prefers-reduced-motion`。

### 2.4 SVG 最佳化

- 上站前跑 **SVGO**（移除註解、多餘小數、metadata）。
- 保留 `viewBox`、拿掉固定 `width/height`（用 CSS 控制）。
- 裝飾性 icon 加 `aria-hidden="true"`；有語意的加 `role="img"` + `<title>`。

## 3. 圖片與插畫策略——真實、有溫度、可信

大牌質感有一半來自**影像**。灰汰郎的優勢是「有真實案例」，要善用。

### 3.1 優先序

1. **真實案例照**（最高優先）：`cases-clean/`、業主另一 repo `Campcool/0988145875` 的 `cases/`。真實 > 一切。
2. **AI 輔助寫實插畫**（僅流程說明）：照 DESIGN.md——必須標「簡易流程說明」、**不得冒充真實案件**、內部資產紀錄要標來源。
3. **授權實拍素材**（補充氛圍）：Unsplash / Pexels（確認授權）或付費庫；**不可**用他牌官網的圖。

### 3.2 讓照片有大牌感（藝術指導）

- **統一色調**：全站照片做一致的色彩分級（白平衡、對比、暖度），避免東一張冷、西一張黃。
- **一致構圖**：留白、主體位置、視角一致；避免雜亂背景。
- **裁切成一致比例**（如 4:3 或 16:10），用 `aspect-ratio` 鎖住避免 CLS。
- **去識別**：客戶臉、門牌、車牌打碼。

### 3.3 讓「AI 插畫」不像 AI（若要用）

- **固定一種風格**（同一種光影、線條、上色邏輯），不要每張風格漂移。
- **合理的工具與姿勢**：手指數量正確、工具握法合理、設備結構正確——AI 圖最常在這裡露餡，要人工挑掉。
- **套品牌色**：後製把畫面主色調拉向灰汰郎色票（深藍/暖橘/青綠），統一氣質。
- **不放畫面內文字**（AI 生的文字會歪）；標題與步驟一律用 HTML。
- **不要誇張髒污/災難感**（DESIGN.md 已規範）。

### 3.4 格式與效能

- 輸出 **AVIF + WebP**（`<picture>` 提供 fallback）；單張流程圖 < 200KB，首頁圖片預算 < 700KB。
- `srcset` + `sizes` 給不同螢幕不同解析度；或 CSS `image-set()`。
- 首屏主圖 `fetchpriority="high"`，其餘 `loading="lazy"` + 明確 `width/height`。
- **blur-up**：先上極小模糊縮圖（base64 佔位），載入後淡入真圖——大牌常見的高級感。

## 4. 研究與素材蒐集技巧（「爬蟲蒐集與利用」的正確作法）

目標是**學到標竿站的「系統」**（配色、字級、間距、動效節奏、資訊順序），再用自己的素材重製——**學模式，不抄素材**。

### 4.1 研究標竿站（合法、只取「作法」不取資產）

- 用瀏覽器 **DevTools** 拆解 apple.com / mi.com / samsung.com / 洗洋洋 官網：
  - **Computed 面板**看真實的 `font-size`、`line-height`、`letter-spacing` → 反推 type scale。
  - **量間距**（padding/gap）→ 反推間距節奏（大牌 section 常 96–160px 垂直留白）。
  - **看 transition/animation** 的 `duration`、`easing`、`transform` → 反推動效節奏。
  - **Network 面板**看圖片格式、大小、載入策略。
- AI/CLI 端可用 **WebFetch / curl** 抓「公開頁面的 HTML/CSS」來研究結構與 class 命名邏輯（僅供學習模式）。
- 用 **Playwright**（本專案已裝，chromium 在 `/opt/pw-browsers/chromium`）：
  - 截圖標竿站不同斷點，做「版型對照牆」。
  - 截自家站同區塊，並排比較差距。
  - 錄捲動過程觀察動效觸發點。

> **版權紅線再強調**：研究＝看它「怎麼排、怎麼動、間距多少」。**不得**下載它的圖片、icon、字型、文案來用。素材一律走 §2.1／§3.1 的授權來源或自製。

### 4.2 蒐集後怎麼「利用」

- 把觀察到的數值**落成 token**（更新 DESIGN.md 的 type scale / 間距 / 動效時間），全站共用。
- 建一份 `docs/REFERENCES.md` 記錄：看了哪些站、學到什麼模式、對應到我們哪個區塊（不放它們的資產，只放文字筆記與自己的截圖）。
- 素材進站前統一過一輪：SVGO（icon）、squoosh/sharp（圖片轉 AVIF/WebP）、字型子集化（只留用到的字）。

### 4.3 工具清單

| 用途 | 工具 |
|---|---|
| SVG 最佳化 | SVGO |
| 圖片壓縮/轉檔 | squoosh-cli、sharp（Node） |
| 版型/動效研究 | 瀏覽器 DevTools、Playwright 截圖 |
| 效能量測 | Lighthouse、WebPageTest（LCP/CLS/TBT） |
| 配色抽取 | DevTools 取色、或把截圖丟色彩分析 |
| 字型子集化 | fonttools / subfont（中文字型很大，務必子集化） |

## 5. 進階互動技法——讓它「有互動」而非呆版

DESIGN.md 的動效規範偏保守（180–320ms、只用 opacity/translate）。要「大牌互動感」可在**尊重 `prefers-reduced-motion`** 的前提下再上一層：

### 5.1 捲動驅動（scroll-driven）
- **IntersectionObserver** 做區塊進場（原型已示範：淡入上滑 + stagger）。
- 新的 **CSS `scroll-timeline` / `animation-timeline`**（漸進增強，不支援就退化為靜態）做視差、進度條、pin 住的產品展示。
- 數字**捲到才 count-up**（價格、服務數）——科技站常見。

### 5.2 招牌互動元件（清潔業超適合）
- **前後對比滑桿**（before/after slider）：拖曳看清洗前 vs 後——對清潔服務極有說服力，比任何文案都強。
- **產品/服務切換器**（原型已做）：切服務換主色、圖、價格、標題。
- **互動估價**：選機型/數量即時算價（可接現有 estimator.js 的 CONFIG）。
- **可篩選案例牆**：依服務/地區篩選真實案例，帶淡入動畫。

### 5.3 微互動與質感細節
- 按鈕/卡片按下的 **spring easing**（`cubic-bezier(.22,.9,.32,1)` 或帶回彈的 `(.34,1.56,.64,1)`）。
- **View Transitions API** 做頁面/狀態切換的平滑轉場（漸進增強）。
- Hover：微浮起 + 色條展開 + icon 輕動（原型已示範），不要劇烈縮放。
- 導覽列捲動**霧化浮出**（backdrop-blur）——原型已做，是很「Apple」的細節。

### 5.4 深度與氛圍（選用、克制）
- 輕量 **parallax**（背景層比前景慢一點）。
- CSS **3D transform** 讓 Hero 元件有厚度；或極輕量 **WebGL/Canvas** 做氣流/水波氛圍（別做成滿版霓虹——那又變 AI 感）。
- **一次只放一個「主秀」**（skill 原則：boldness 花在一個地方，其餘保持安靜）。

## 6. 字體與排版精修

- **放大、自信**：Hero 標題用 `clamp(2.4rem, 6vw, 4.2rem)`、負字距（`letter-spacing:-.02em`）、`text-wrap:balance`。
- **中文字型子集化**：Noto Sans TC 全字檔很大，務必子集化或用 `unicode-range` 分包，改善 LCP。
- **等寬數字**：價格、坪數、電話用 `font-variant-numeric:tabular-nums`——規格表般的精準感。
- **行寬控制**：內文最大約 60–65 全形字（`max-width: ~34ch`）。
- **標籤/eyebrow**：小字 + 字距（`letter-spacing:.16em`）+ 大寫/專屬色，結構感來自這裡。

---

## 7. 效能與無障礙（大牌的隱形門檻）

- **LCP < 2.5s、CLS < 0.1**：圖片鎖 `aspect-ratio`、字型 `font-display:swap` + 子集化、首屏圖 `fetchpriority=high`。
- 手機 360px 寬**不得水平捲動**。
- **鍵盤 focus 可見**、對比 ≥ WCAG AA、裝飾圖 `alt=""`、語意結構正確。
- **一律** `@media (prefers-reduced-motion: reduce)` 關動效。
- header runtime 注入造成的 CLS（AI-README §7 已列）要處理：用等高 spacer 佔位。

## 8. 具體可加的「技能/做法」清單（給 Codex 慢慢琢磨）

依投入報酬排序，Codex 可逐項導入：

1. **移除所有 emoji → 換品牌化 icon sprite**（§2）。最高優先，立刻去 AI 感。
2. **前後對比滑桿**放到冷氣/洗衣機/清潔頁（§5.2）。轉換率殺手級。
3. **捲動進場 + 服務切換器**全站化（§5.1）。
4. **照片色調統一 + AVIF/WebP + blur-up**（§3）。
5. **type scale 放大、子集化中文字型**（§6）。
6. **Hero 重做**：拿掉藍紫漸層，改留白 + 大字 + 一個主秀互動（§1、§5.4）。
7. **卡片改邊框為主、陰影只在 hover**（DESIGN.md 已規範）。
8. **互動估價接 estimator.js**（§5.2）。
9. **count-up 數字、View Transitions、輕量 parallax**（§5）。
10. **REFERENCES.md 研究筆記**持續累積（§4.2）。
