## ⚠️ 動工前的唯一前提：轉換追蹤（沒有它就別開廣告）

小預算更輸不起浪費，一定要能算轉換。順序：

1. **GA4**：確認/建立 GA4 資源，取得真實評估 ID `G-XXXXXXXXXX`。
2. **填 ID**：把該 ID 填進網站 `header.js` 的 `GA4_ID` 常數（目前是佔位值，3 處）。→ 此步由 **Claude/Codex 在 repo 做**，需要業主先給真實 ID。
3. **GA4 標記關鍵事件**：`quote_submit`＝**主要轉換**、`line_click`＝次要轉換。
4. **Google Ads 匯入轉換**：Google Ads → 「工具 → 資料管理 → Google Analytics (GA4)」連結資源 → 匯入 `quote_submit`（設為**主要**）、`line_click`（設為**次要/僅觀察**）。
5. **驗證**：到網站送一次測試表單，GA4 即時報表 + Google Ads 轉換診斷要看得到。
6. **出價目標** = `quote_submit`。

> 若上線當下追蹤還沒好：出價策略先用「盡量爭取點擊（設出價上限）」，等轉換進來再切智慧出價（見 §4）。

## 2. 活動層級設定（每個活動都這樣設）

- **活動類型**：搜尋（Search）。目標可選「潛在客戶」。
- **網路**：✅ 搜尋聯播網；**關閉「顯示廣告聯播網」**（會偷吃預算）；搜尋夥伴可先開、觀察。
- **地區**：台北市、新北市、基隆市、桃園市、新竹市、新竹縣、苗栗縣、台中市。
  - 地區選項務必選「**目標對象的所在地**（實際位於或經常位於）」，**不要**選「有興趣」。
- **語言**：中文（繁體）。
- **裝置**：全部；行動裝置為主客群（可不調，先看數據）。
- **出價策略**：**Phase 1 用「盡量爭取點擊」＋設出價上限**（見各活動 §4）。累積 ≥15 轉換/月後改「盡量爭取轉換」或「目標 CPA」。
- **廣告輪替**：最佳化（預設）。
- **最終到達網址後綴（追蹤模板，整個活動貼一次）**：
  ```
  utm_source=google&utm_medium=cpc&utm_campaign={campaignid}&utm_content={adgroupid}&utm_term={keyword}
  ```
