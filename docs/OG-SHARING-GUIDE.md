# 灰汰郎知識型社群分享規範

## 核心原則

社群預覽先回答使用者的問題，不先推銷服務。使用者點進網站後，再透過頁面內容、案例與詢問按鈕完成轉換。

- 標題採用問題句或判讀句，例如「冷氣為什麼會漏水？」
- 描述交代讀者會學到的原因、流程或判斷方式。
- 圖片以剖面、流程、成因或前後差異為主，不放價格、促銷章、LINE CTA 或「立即預約」。
- 圖片與文字不得暗示 AI 流程示意是真實完工案例。
- `og:image` 與 `twitter:image` 必須一致，並提供 `og:image:alt`。

## 新頁面檢查表

1. 每個可分享頁面使用自己的 OG 圖片網址，避免社群快取互相污染。
2. `og:title` 不使用「到府服務」「優惠」「立即預約」作為主標。
3. `og:description` 不放價格與促銷話術，專注原因、流程、案例或自我檢查。
4. 圖片保持橫式社群卡片構圖，重要內容避開四周裁切區。
5. 上線後使用 Facebook Sharing Debugger 與 LINE 實際貼文重新抓取驗證。
