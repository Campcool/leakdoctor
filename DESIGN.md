# 灰汰郎 Design System

最後更新：2026-07-11（Codex）

## 1. 品牌定位

灰汰郎不是冷冰冰的報價平台，也不是以低價吸引點擊的清潔電商。品牌應像一位熟悉居家問題、願意先把事情說清楚的服務顧問。

核心感受：

- 專業但不壓迫
- 溫暖但不幼稚
- 清楚但不制式
- 有效率但不催促
- 以真實流程建立信任，不使用誇大的「神奇前後對比」

對外文字以「我們」與「服務團隊」為主；施工由合作廠商執行時須清楚說明安排角色。

## 2. 視覺氣氛

設計融合溫暖的居家編輯感、服務業的可信任感與精準的資訊層級。避免常見 AI 網站特徵：大面積霓虹漸層、玻璃擬態、過量 emoji、每張卡片都帶陰影、空泛的「智慧解決方案」文案。

圖片原則：

- 優先使用真實案例；教育流程可使用 AI 輔助的寫實編輯插畫。
- AI 輔助圖片不得冒充實際施工案例；對外統一標示「簡易流程說明」，並在內部資產紀錄保留來源。
- 工具、保護措施與操作姿勢必須合理。
- 不呈現誇張髒污、災難式漏水或恐嚇情境。
- 圖片內不放文字，標題與步驟由 HTML 呈現，確保手機可讀與 SEO。

### 同業操作參考

可參考合作夥伴洗洋洋官網的資訊順序：首屏快速理解服務與價格、服務分類、流程 Step、真實案例、FAQ、LINE 預約。灰汰郎只採用操作邏輯，不複製其品牌配色、圖片、版型或文案。

灰汰郎保留的差異：

- 主導覽維持四大服務，不展開成大量分類選單。
- 每頁先提供問題判讀與服務邊界，不只呈現促銷價格。
- 不使用無證據的健康、殺菌、保固、最低價或好評數字宣稱。
- 聯絡入口維持 LINE，不恢復公開電話 CTA。
- 流程固定以確認、斷電／停水、保護、處理、復原、測試為主。

## 3. 色彩

| Token | 色碼 | 用途 |
|---|---|---|
| `--ink` | `#17324D` | 標題、主要導覽、深色區塊 |
| `--ink-deep` | `#10273B` | Footer、深色互動狀態 |
| `--orange` | `#F28C28` | 主要 CTA、價格重點、品牌識別 |
| `--orange-dark` | `#D96F0D` | CTA hover/active |
| `--teal` | `#138A80` | 完成、安心、檢測與服務狀態 |
| `--line` | `#06C755` | 僅限 LINE 官方操作 |
| `--cream` | `#FFF9F1` | Hero 與溫暖背景 |
| `--sand` | `#F3E8D8` | 分隔、柔和底色 |
| `--paper` | `#FFFFFF` | 卡片與內容底 |
| `--text` | `#263238` | 內文 |
| `--muted` | `#667680` | 次要文字 |
| `--border` | `#DCE4E7` | 邊框 |

規則：單一畫面最多使用一個主 CTA 色。LINE 綠只代表「開啟 LINE」，不能成為通用成功色。

## 4. 字體與層級

主要字體：`Noto Sans TC`。暫不依賴額外襯線字體，避免載入與跨裝置差異；品牌編輯感由字級、字重與行寬建立。

| 樣式 | 桌機 | 手機 | 字重 |
|---|---:|---:|---:|
| Display | 56px | 38px | 900 |
| H1 | 48px | 34px | 900 |
| H2 | 34px | 28px | 900 |
| H3 | 22px | 20px | 800 |
| Body L | 18px | 17px | 400 |
| Body | 16px | 16px | 400 |
| Label | 13px | 13px | 700 |

- 標題行高 1.15–1.3；內文行高 1.7–1.9。
- 內文單行最大約 62 個中文字元。
- 不以全形驚嘆號或連續粗體製造壓迫感。

## 5. 間距與版面

使用 4px 基準：`4 / 8 / 12 / 16 / 24 / 32 / 48 / 64 / 96`。

- 內容最大寬度：1200px。
- 閱讀內容最大寬度：760px。
- 桌機 section 垂直留白：80–104px。
- 手機 section 垂直留白：56–72px。
- 手機左右安全邊距：20px；桌機：32px。
- 卡片內容邊距：24–32px。

首頁每個區塊只回答一個問題：這是什麼、能解決什麼、怎麼做、多少錢、為何可信、如何開始。

## 6. 元件

### 按鈕

- Primary：品牌橘底、白字、14px 圓角；用於「開始估價／預約」。
- LINE：LINE 綠底、白字；文字必須明確寫「LINE 諮詢」。
- Secondary：透明或白底、深藍字、至少 1.5px 邊框並帶低對比陰影。
- 高度至少 48px，手機觸控區至少 44×44px。
- Hover 只做顏色、陰影和 1–2px 位移，不做劇烈縮放。

### 卡片

- 圓角 20–24px。
- 預設使用邊框，不依賴陰影切割資訊。
- 同一區塊最多一層卡片嵌套。
- 卡片 hover 不應隱藏必要資訊。

### 表單

- Label 永遠顯示，不以 placeholder 代替。
- 欄位高度至少 48px。
- 錯誤訊息說明如何修正，不只顯示紅框。
- 預約前說明資料用途與下一步。
- 四大服務不得使用單一原生下拉選單；使用 2 欄服務卡，顯示對應 SVG、名稱與明確選取狀態。
- 日期與時段可以保留原生控制，維持手機系統操作習慣。

### 導覽與圖示

- 主導覽固定四項，手機 2×2、桌機 4 欄；卡片尺寸與 Hero 起點在各頁一致。
- 桌機 Header 採單列五入口：Logo 即首頁入口，與四個服務頁籤同為 72px 高並置中對齊；服務圖示 30px、主標 15px，確保寬螢幕能立即辨識與切換。
- Header 固定於視窗頂部，捲動內容時 Logo 與四個頁籤不可離開畫面；使用等高 spacer 保留內容起點，不以手寫固定留白猜測高度。
- 四服務採專屬色彩並延伸至頁面 CTA 與重點區：冷氣青藍、洗衣機紫、居家清潔琥珀、漏水檢測青綠；非作用中頁籤仍保留自己的圖示色，作用中頁籤使用完整底色。
- 公開前台不顯示「加入我們」合作招募入口，主要浮動操作只保留 LINE。
- 圖示使用同一組 24px 線性 SVG：冷氣室內機、洗衣機、住宅清潔、漏水檢測；不得再用不一致的 emoji 代替服務類型。
- Header 不另放目的不明的知識按鈕；知識內容整合在四大服務頁的「先了解，再決定」區塊及內容內連結。
- Logo 在 360px 手機仍需具品牌可讀性，不可被右側操作壓縮成小圖章。
- 公開聯絡操作以 LINE 為主，不顯示電話 CTA；預約表單仍需蒐集派工聯絡電話。
- Header 本體只使用細邊界，不使用外陰影；服務選項本身可使用邊框與低對比陰影表示可點擊。
- LINE 使用右側懸浮按鈕，每 2 秒一次柔和脈衝；不得遮住底部預約列或回頂按鈕。
- 手機系統返回鍵在預約抽屜開啟時，必須先關閉抽屜，不可直接離開頁面。

### 狀態標籤

- 橘：需要選擇或注意。
- 青綠：已確認、可服務。
- 深藍：一般資訊。
- 紅色只用於失敗或高風險，不用於促銷倒數。

## 7. 首頁 UX 順序

1. Hero：一句清楚承諾、主要 CTA、次要 CTA、真實服務情境。
2. 信任列：價格透明、先確認再施工、合作團隊安排。
3. 服務選擇：四大服務與起價/報價方式。
4. 簡易流程說明：冷氣、洗衣機、居家清潔、抓漏各自以六步驟說明如何處理。
5. 快速估價器：讓已有意圖的訪客直接取得下一步。
6. 服務保障與實際案例。
7. 服務地區與最終 CTA。

廣告落地頁不得把「加入廠商」放在主要轉換路徑附近。

## 8. 動效

- 尊重 `prefers-reduced-motion`。
- 進場動畫 180–320ms，僅使用 opacity/translate。
- 不使用無限脈衝 CTA；必要提示最多播放一次。
- 捲動時固定元件不得遮住表單、Cookie 或系統鍵盤。

## 9. 無障礙與效能

- 文字對比至少 WCAG AA。
- 圖片提供具體 alt；純裝飾圖片使用空 alt。
- 鍵盤 focus 必須可見。
- 流程圖片使用 WebP/AVIF、明確寬高與 lazy loading。
- 首屏圖片可 `fetchpriority="high"`，其餘 `loading="lazy"`。
- 單張流程圖建議低於 200 KB；首頁總圖片預算建議低於 700 KB。
- 手機 360px 寬不得產生水平捲動。

## 10. 文案規則

使用具體語句：

- 「先確認機型與現場條件，再提供報價」
- 「送出後開啟 LINE，由服務團隊確認」
- 「需要專員聯絡協處」

避免：

- 「AI 智慧安排」、「一鍵解決所有問題」
- 「全台第一」、「最低價」、「保證有效」等無證據宣稱
- 假倒數、假名額、假即時預約通知
- 把簡易流程圖說成真實施工案例

## 11. 廣告投放前門檻

- GA4 真實 Measurement ID 已設定。
- `quote_open`、`quote_submit`、`line_click` 可在 DebugView 驗證。
- 每個廣告群組對應單一服務落地頁，不全部導向首頁。
- 價格、服務地區與 LINE 連結一致。
- 手機 LCP、CLS、表單完成率完成基準量測。
- 簡易流程說明與真實案例分區呈現，內部資產紀錄須標記 AI 圖片來源。
